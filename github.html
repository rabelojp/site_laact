GitHub
(hospedagem de código)

O que é o GitHub?

GitHub é uma plataforma de hospedagem de código-fonte que utiliza o sistema de controle de versão Git. Ela permite que desenvolvedores e equipes colaborem em projetos de software, gerenciem o código, rastreiem mudanças e compartilhem seus trabalhos.

Criar uma conta no GitHub

Acesse o site

Acesse o site oficial do GitHub (https://github.com/) no canto superior direito, clique em Sign up para iniciar o processo de cadastro.

Informações

Insira seu e-mail, crie uma senha e um nome de usuário da estar criando sua conta no GitHub.

Confirmação

Informe o código de 8 dígitos enviado para estar confirmando seu e-mail na plataforma.

Tudo pronto

Depois de finalizar as últimas configurações, você vai ser direcionado a uma página parecida com essa, onde terá acesso a diversas configurações.

Crie seu primeiro repositório

Agora que sua conta está configurada e o Git está instalado, vamos criar um repositório no GitHub.

Atenção: é de suma importância que antes de criar o repositório você tenha baixado e instalado o Git no seu computador e tenha configurado seu nome e e-mail no sistema.

Criando o repositório

No canto superior esquerdo da página inicial, clique em + e depois em New repository (novo repositório)

Definindo o repositório

Owner:Selecione seu nome de usuário. Este será o "dono" do repositório e quem terá controle total sobre ele.

Repository name: Escolha um nome para o seu repositório (exemplo: meu-projeto). Este nome deve ser único dentro da sua conta do GitHub e geralmente é em letras minúsculas, sem espaços (use hífen ou underline para separar palavras, como meu_projeto ou meu-projeto).

Description: Insira uma descrição curta do seu projeto. A descrição ajuda a explicar o propósito do repositório e fornecer um breve contexto sobre o que o projeto faz.

Defina se o repositório será público ou privado:

Público: : Qualquer pessoa pode ver esse repositório e contribuir (se permitido);

Privado:Apenas você e pessoas que você convidar poderão acessar o repositório. Ideal para projetos pessoais ou que não estão prontos para serem divulgados.

Readme: um documento que serve para dar uma visão geral do projeto. Geralmente, ele contém informações como:

  • • Descrição do projeto;
  • • Instruções de instalação e configuração;
  • • Exemplos de uso;
  • • Como contribuir para o projeto.

Ao marcar esta opção, o GitHub criará automaticamente um arquivo README.md no repositório com um texto básico. Você pode editar esse arquivo depois para adicionar mais informações.

.gitignore: É importante configurar o .gitignore para evitar que arquivos indesejados sejam incluídos no repositório, o que pode aumentar o tamanho desnecessariamente e expor informações sensíveis.

Choose a license: Uma licença define os direitos e permissões que outras pessoas têm ao usar, modificar ou distribuir seu código.

Create repository: ao terminar a configuração clique em Create Repository para estar criando.
